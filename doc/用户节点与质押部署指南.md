# 用户节点与质押部署指南

本文档描述用户下载项目后从启动节点到成为验证者的最小闭环。

## 1. 构建二进制

```powershell
go build -o .\dist\posnode.exe .\cmd\posnode
go build -o .\dist\wallet.exe .\cmd\wallet
```

## 2. 生成验证者密钥

```powershell
.\dist\wallet.exe validator-keygen -out-dir .\keys\validator-node
```

命令会生成：

- `peer.json`：P2P 身份密钥
- `staker.json`：质押资金账户密钥
- `validator.json`：验证者账户密钥
- `consensus.json`：共识投票密钥
- `bls.json`：BLS 聚合签名密钥

终端会输出 `peer_id`、`validator_address`、`consensus_address`、`bls_public_key`。这些是公开身份，可以提交到注册验证者交易。不要公开 `.json` 密钥文件内容。

## 3. 配置并启动节点

在节点配置中设置密钥路径：

```json
{
  "production": true,
  "node_role": "validator",
  "node_capabilities": ["validator", "state_sync", "dht"],
  "peer_key_path": "keys/validator-node/peer.json",
  "staker_key_path": "keys/validator-node/staker.json",
  "validator_key_path": "keys/validator-node/validator.json",
  "consensus_key_path": "keys/validator-node/consensus.json",
  "bls_key_path": "keys/validator-node/bls.json",
  "rpc_enabled": true,
  "rpc_listen_ip": "0.0.0.0",
  "rpc_port": 8899
}
```

启动：

```powershell
.\dist\posnode.exe -config .\config\validator.json
```

启动日志会显示配置文件路径、密钥文件位置、公钥地址和 PeerID。日志不会显示 seed 或私钥值。

## 4. 查询收款地址和余额

```powershell
.\dist\wallet.exe address -key .\keys\validator-node\staker.json
.\dist\wallet.exe balance -rpc http://127.0.0.1:8899 -address <STAKER_ADDRESS>
```

创世账户或水龙头给 `STAKER_ADDRESS` 转账后，用户即可发起验证者注册。

## 5. 注册验证者

```powershell
.\dist\wallet.exe validator-register `
  -rpc http://127.0.0.1:8899 `
  -staker-key .\keys\validator-node\staker.json `
  -validator-address <VALIDATOR_ADDRESS> `
  -consensus-address <CONSENSUS_ADDRESS> `
  -bls-public-key <BLS_PUBLIC_KEY> `
  -peer-id <PEER_ID> `
  -lamports 10000000
```

注册交易由钱包本地签名，再通过公网 RPC 的 `sendTransaction` 广播。不要把 seed 传给 RPC。

## 6. 普通用户委托质押

普通用户可以通过钱包查看 `getValidatorSet` 返回的验证者列表，选择验证者账户后发起 DPoS 委托质押。

```powershell
.\dist\wallet.exe delegate `
  -rpc http://127.0.0.1:8899 `
  -key .\keys\user.json `
  -validator-address <VALIDATOR_ADDRESS> `
  -lamports 10000000
```

委托质押会进入验证者 stake account 的委托列表，下一 epoch 后参与验证者权重和收益分配。

取消委托：

```powershell
.\dist\wallet.exe undelegate `
  -rpc http://127.0.0.1:8899 `
  -key .\keys\user.json `
  -validator-address <VALIDATOR_ADDRESS> `
  -lamports 10000000 `
  -unlock-epoch <UNLOCK_EPOCH>
```

提现已解锁委托：

```powershell
.\dist\wallet.exe withdraw-delegation `
  -rpc http://127.0.0.1:8899 `
  -key .\keys\user.json `
  -validator-address <VALIDATOR_ADDRESS> `
  -current-epoch <CURRENT_EPOCH>
```

奖励分配规则：

- 验证者先按 `commission_bps` 收取佣金
- 剩余奖励按 active 自质押和 active 委托质押份额自动分配
- pending 委托下一 epoch 生效后才参与收益
- 奖励结算在 epoch settlement 中由链上共识状态转换自动执行

## 7. 验证者追加自质押

```powershell
.\dist\wallet.exe stake `
  -rpc http://127.0.0.1:8899 `
  -staker-key .\keys\validator-node\staker.json `
  -validator-address <VALIDATOR_ADDRESS> `
  -lamports 10000000
```

`stake` 用于验证者 staker 对自己的验证者账户追加自质押；普通用户委托应使用 `delegate`。

## 8. 查看验证者集合

```powershell
$body = @{
  jsonrpc = "2.0"
  id = 1
  method = "getValidatorSet"
  params = @()
} | ConvertTo-Json

Invoke-RestMethod -Uri http://127.0.0.1:8899 -Method Post -ContentType "application/json" -Body $body
```

注册或追加质押后，stake 会按 epoch 规则生效。
