# 项目结构体

版本：v1  
范围：`structure` 包内核心事实模型、P0 执行输入输出模型、区块查询视图和基础运行时上下文。

## 一、设计边界

1. `structure` 包只定义事实模型、边界校验、确定性序列化和安全拷贝。
2. `structure` 包不直接读写数据库，不启动网络，不执行共识状态机。
3. 账户状态变更必须由后续执行器产生写集，再由数据库事务提交。
4. P2P、数据库 raw payload、链上账户 data、instruction data 使用 Borsh 或固定字段二进制布局。
5. 外部 RPC 只能通过 DTO 转换，不得直接暴露内部可变切片。

## 二、值对象

| 结构体/类型 | 文件 | 作用 |
| --- | --- | --- |
| `PublicKey` | `structure/value.go` | 32 字节账户地址值对象。 |
| `Hash` | `structure/value.go` | 32 字节哈希值对象。 |
| `Blockhash` | `structure/value.go` | 区块哈希别名。 |
| `TransactionHash` | `structure/value.go` | 交易摘要别名。 |
| `Signature` | `structure/value.go` | 64 字节 Ed25519 签名值对象。 |
| `SolanaKeyPair` | `structure/keypair.go` | 公钥和 32 字节私钥 seed 的密钥材料。 |
| `SignatureVerification` | `structure/signature.go` | 单个签名校验结果。 |

说明：

1. `PublicKey` 全零值是有效地址，不再作为空值拒绝。
2. 所有 `Bytes()` 方法必须返回拷贝，禁止暴露内部数组或切片。
3. 私钥、seed、secret key 不得进入日志、RPC 响应或 P2P 消息。

## 三、账户结构

| 结构体 | 文件 | 作用 |
| --- | --- | --- |
| `RentConfig` | `structure/account.go` | 租金参数配置。 |
| `Account` | `structure/account.go` | 链上账户 value，包含余额、数据、owner、可执行标记和租金 epoch。 |
| `AccountMeta` | `structure/transaction.go` | 交易消息中的账户权限元数据。 |
| `AddressedAccount` | `structure/loaded_transaction.go` | 地址和账户 value 的绑定，用于数据库和执行写集。 |

`Account` 字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `Lamports` | `uint64` | 账户余额。 |
| `Data` | `[]byte` | 账户数据，必须拷贝读写。 |
| `Owner` | `PublicKey` | 有权修改 data 和扣减余额的程序地址。 |
| `Executable` | `bool` | 是否为可执行程序账户。 |
| `RentEpoch` | `uint64` | 租金相关 epoch。 |

租金规则：

1. 默认最小余额公式：`(data_len + 128) * 3480 * 2`。
2. 新建账户、扣减余额、扩容 data 后必须重新校验租金豁免。
3. 账户 data 最大 10 MiB。
4. 单次指令 data 增长最大 10 KiB。
5. 关闭账户时应删除账户 key，并把 lamports 转入目标账户，不保存零余额脏状态。

## 四、交易消息结构

| 结构体/类型 | 文件 | 作用 |
| --- | --- | --- |
| `TransactionStatus` | `structure/transaction.go` | 交易状态枚举。 |
| `MessageHeader` | `structure/transaction.go` | 账户签名和只读权限计数。 |
| `CompiledInstruction` | `structure/transaction.go` | 交易消息中的紧凑指令。 |
| `MessageAddressTableLookup` | `structure/transaction.go` | v0 地址表引用。 |
| `SolanaMessage` | `structure/transaction.go` | 可签名交易消息。 |
| `Transaction` | `structure/transaction.go` | 交易本体，包含签名和消息。 |

`Transaction` 字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `Signatures` | `[]Signature` | 交易签名列表。 |
| `Accounts` | `[]AccountMeta` | 兼容构建消息时的账户权限列表。 |
| `Instructions` | `[]CompiledInstruction` | 已编译指令列表。 |
| `RecentBlockhash` | `Blockhash` | recent blockhash。 |
| `Message` | `SolanaMessage` | 显式交易消息。 |
| `Fee` | `uint64` | 已计算费用缓存。 |
| `Size` | `uint64` | 序列化大小缓存。 |
| `SubmitTime` | `int64` | 进入交易池时间。 |
| `Status` | `TransactionStatus` | 当前交易状态。 |

核心规则：

1. 交易最大序列化大小 1232 字节。
2. 交易 ID 使用第一笔签名。
3. 交易摘要使用完整 wire bytes 计算。
4. 签名数量必须等于 `MessageHeader.NumRequiredSignatures`。
5. 账户顺序必须按签名/可写权限分组。

1. `CreateAccount`
2. `Transfer`
3. `Assign`
4. `Allocate`

边界：

1. `SystemInstruction` 只定义 instruction data。
2. 它不直接扣款、不创建账户、不写数据库。
3. 账户状态转换由后续执行器完成。

## 七、费用和计算预算

| 结构体 | 文件 | 作用 |
| --- | --- | --- |
| `ComputeBudgetLimits` | `structure/fee.go` | 交易计算资源上限。 |
| `PrioritizationFee` | `structure/fee.go` | 优先费输入。 |
| `FeeDetails` | `structure/fee.go` | 费用明细。 |
| `FeeCalculator` | `structure/fee.go` | 费用计算参数。 |

默认规则：

1. 基础签名费：`5000 lamports/signature`。
2. 优先费：`ceil(compute_unit_limit * compute_unit_price_micro_lamports / 1_000_000)`。
3. 单交易最大计算单元：`1_400_000`。
4. 默认加载账户 data 上限：`64 MiB`。
5. 基础费默认 50% 燃烧，剩余给当前出块者。

| `TransactionSimulationInput` | `structure/simulator.go` | 交易模拟执行输入。 |
| `TransactionSimulator` | `structure/simulator.go` | 内存交易模拟执行器。 |

`LoadedTransaction` 字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `Transaction` | `Transaction` | 原始交易快照。 |
| `Message` | `ResolvedMessage` | 执行期账户索引表。 |
| `Accounts` | `[]LoadedAccount` | 已加载账户快照。 |

校验规则：

1. `LoadedAccount` 数量必须等于 `ResolvedMessage.AccountKeys` 数量。
2. 每个加载账户地址必须和消息账户表同位置一致。
3. `FeePayer` 必须等于第 0 个账户。
4. 地址表解析后账户不能重复。

## 九、交易模拟执行

| 结构体 | 文件 | 作用 |
| --- | --- | --- |
| `TransactionSimulationInput` | `structure/simulator.go` | 组合交易、账户快照、blockhash 队列、slot、费用、租金和内置程序配置。 |
| `TransactionSimulator` | `structure/simulator.go` | 在内存中模拟交易执行并返回执行结果。 |

模拟执行流程：

1. 校验交易结构。
2. 校验交易签名。
3. 校验 recent blockhash 是否仍在有效窗口。
4. 解析交易消息和账户索引。
5. 加载输入账户快照。
6. 计算 fee 并从 fee payer 扣除。
7. 顺序执行系统指令。
8. 生成 `TransactionExecutionResult`。
9. 返回账户写集，不直接提交数据库。

当前支持的系统指令：

1. `Transfer`
2. `CreateAccount`
3. `Assign`
4. `Allocate`

模拟执行边界：

1. 模拟器只使用内存账户快照，不读取数据库。
2. 模拟器返回 `WrittenAccounts`，调用方决定是否提交。
3. 交易级失败返回 `Status == Failed` 的结果，不作为 Go error。
4. Go error 只表示模拟器内部状态不一致或结果无法自校验。
5. 当前不执行用户程序、地址表账户解析、nonce 账户和 loader 程序。

| `PostTokenBalances` | `[]TokenBalance` | 执行后 token 余额。 |

规则：

1. `Status == Failed` 时必须有 `Error`。
2. 非失败状态不得携带 `Error`。
3. `PreBalances` 和 `PostBalances` 长度必须一致。
4. 写集账户必须通过账户校验。

## 十二、运行时上下文结构

| 结构体 | 文件 | 作用 |
| --- | --- | --- |
| `ClockSysvar` | `structure/sysvar.go` | slot、epoch、unix 时间上下文。 |
| `RentSysvar` | `structure/sysvar.go` | 租金上下文。 |
| `InstructionsSysvar` | `structure/sysvar.go` | 当前交易指令上下文。 |

说明：

1. sysvar 是程序读取执行上下文的模型。
2. P0 只定义结构和校验，不负责将 sysvar 注入账户存储。
3. `InstructionsSysvar.CurrentInstruction()` 返回拷贝，禁止程序修改原始交易指令。

## 十三、blockhash 和状态缓存

| 结构体 | 文件 | 作用 |
| --- | --- | --- |
| `RecentBlockhashEntry` | `structure/blockhash_status.go` | recent blockhash 记录。 |
| `BlockhashQueue` | `structure/blockhash_status.go` | 有界 recent blockhash 队列。 |
| `StatusCacheEntry` | `structure/blockhash_status.go` | 交易状态缓存项。 |

规则：

1. recent blockhash 默认有效窗口为 150 slots。
2. `BlockhashQueue.Add()` 会自动裁剪过期记录。
3. 状态缓存禁止保存 `Pending` 状态。
4. 状态缓存必须至少包含交易 ID 或消息 hash。

1. 系统程序地址是全零公钥。

## 十五、后续缺口

P0 当前已支持内存交易模拟执行。后续还需要：

1. 数据库提交：用事务原子提交账户写集、交易状态、区块索引。
2. 地址查找表账户：补完整数据结构和解析逻辑。
3. Nonce 账户：支持离线签名和 recent blockhash 替代。
4. Loader 程序账户：支持程序部署、升级和执行。
5. Vote/Stake 账户：支持验证者投票和权益权重。
6. 用户程序执行器：接入 VM 或原生程序分发。
7. RPC DTO：将内部结构转换为外部 JSON 视图。
