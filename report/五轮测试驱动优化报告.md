# 五轮测试驱动优化报告

## 测试范围

本轮聚焦当前本机可验证的生产门禁和多节点巡检工具，不声明已完成真实 5 机/30 节点长稳验证。真实网络验证仍需要节点全部启动后提供 RPC 地址文件。

## 第一轮：生产门禁参数冲突

### 原始测试

命令：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\run_posnode_gate.ps1 -SkipRace -RequireRace
```

问题输出：

```text
==> go test .\...
==> go vet .\...
==> go build .\cmd\posnode
WARNING: race gate skipped by -SkipRace. Production release must run go test -race on Linux or macOS.
```

### 问题分析

`-SkipRace` 和 `-RequireRace` 是互斥语义。原逻辑优先处理 `-SkipRace`，导致调用方同时传入两个参数时 race 门禁被跳过。这会让生产发布脚本出现错误放行风险。

### 优化内容

- 为 `tools/run_posnode_gate.ps1` 增加 `-Help`。
- 增加 `-SkipRace` 与 `-RequireRace` 冲突校验。
- 冲突时直接退出，避免执行耗时测试并错误跳过 race。

### 复测结果

命令：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\run_posnode_gate.ps1 -SkipRace -RequireRace
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\run_posnode_gate.ps1 -Help
```

关键输出：

```text
-SkipRace and -RequireRace cannot be used together
Usage:
  powershell -ExecutionPolicy Bypass -File .\tools\run_posnode_gate.ps1 [-SkipRace]
  powershell -ExecutionPolicy Bypass -File .\tools\run_posnode_gate.ps1 -RequireRace
```

结论：第一轮通过，生产门禁参数误用不再被静默放行。

## 第二轮：巡检参数边界校验

### 原始测试

命令：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -Rounds 0 -RpcUrls http://127.0.0.1:1
```

问题输出：

```text
consistency check passed
```

### 问题分析

`Rounds=0` 时脚本没有执行任何采样，却返回成功。这会让长稳巡检在参数传错时生成假阳性结论，属于生产验证工具不可接受的问题。

### 优化内容

为 `tools/check_posnode_consistency.ps1` 增加参数边界校验：

- `Rounds >= 1`
- `IntervalSeconds >= 0`
- `TimeoutSeconds >= 1`
- `MaxHeightDrift >= 0`

### 复测结果

命令：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -Rounds 0 -RpcUrls http://127.0.0.1:1
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -IntervalSeconds -1 -RpcUrls http://127.0.0.1:1
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -TimeoutSeconds 0 -RpcUrls http://127.0.0.1:1
```

关键输出：

```text
Rounds must be greater than or equal to 1
IntervalSeconds must be greater than or equal to 0
TimeoutSeconds must be greater than or equal to 1
```

结论：第二轮通过，巡检脚本不会再因为非法轮数产生假成功。

## 第三轮：失败巡检报告落盘

### 原始测试

命令：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -RpcUrls http://127.0.0.1:1 -OutputPath .\report\tmp\missing-dir\consistency.json -Rounds 1 -TimeoutSeconds 1
```

问题输出：

```text
Set-Content : Could not find a part of the path 'F:\workSpace2029\solana_golang\report\tmp\missing-dir\consistency.json'
```

### 问题分析

真实多节点测试中，节点不可达、网络抖动、leader 离线都可能触发失败。失败时更需要保留 JSON 诊断报告。原脚本不会创建输出目录，导致错误样本已经采集后无法落盘。

### 优化内容

- 新增 `Write-JsonReport`。
- 写报告前自动解析输出路径并创建父目录。
- 成功和失败场景都使用同一个 JSON 写入路径。

### 复测结果

命令：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -RpcUrls http://127.0.0.1:1 -OutputPath .\report\五轮测试输出\round3-consistency.json -Rounds 1 -TimeoutSeconds 1
Get-ChildItem -Path .\report -Recurse -Filter round3-consistency.json
```

关键输出：

```text
==> consistency round 1/1
http://127.0.0.1:1 snapshot failed: 操作超时
round 1: round 1 has no successful node snapshot
F:\workSpace2029\solana_golang\report\五轮测试输出\round3-consistency.json
```

结论：第三轮通过，失败巡检也能保留结构化原始证据。

## 第四轮：巡检字段缺失容错

### 原始风险

巡检脚本开启了：

```powershell
Set-StrictMode -Version Latest
```

但脚本中存在直接读取 JSON 字段的逻辑，例如：

```powershell
$status.consensus
$Status.transaction_fast_path.leader_slots
$Snapshot.Status.transaction_fast_path.start_slot
```

### 问题分析

当节点版本不一致、RPC 字段尚未升级或业务字段临时缺失时，StrictMode 会把缺失属性当作脚本错误，而不是业务不一致错误。这会降低多节点巡检工具的可用性，尤其不利于滚动升级和混部测试。

### 优化内容

- 新增 `Get-PropertyValue`、`Get-ArrayValue`、`Get-StringProperty`、`Get-UInt64Property`。
- 对 `consensus`、`transaction_fast_path`、validator 列表、Turbine 字段、leader 字段统一使用安全读取。
- 缺少 `transaction_fast_path` 时转为明确业务错误：`missing transaction_fast_path`。
- 缺少 `consensus` 时仍尝试调用独立 `getConsensusStatus`。

### 复测结果

命令：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -Help
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -RpcUrls http://127.0.0.1:1 -OutputPath .\report\五轮测试输出\round4-unreachable.json -Rounds 1 -TimeoutSeconds 1
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -MaxHeightDrift -1 -RpcUrls http://127.0.0.1:1
```

关键输出：

```text
Usage:
  powershell -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -RpcUrls http://127.0.0.1:5300,http://127.0.0.1:5301 -Rounds 12 -IntervalSeconds 5
http://127.0.0.1:1 snapshot failed: 操作超时
round 1: round 1 has no successful node snapshot
MaxHeightDrift must be greater than or equal to 0
F:\workSpace2029\solana_golang\report\五轮测试输出\round4-unreachable.json
```

结论：第四轮通过，脚本基础路径未回归，字段缺失已改为可控业务校验路径。

## 第五轮：脚本自检与退出码一致性

### 原始风险

巡检脚本没有离线自检模式。没有真实节点时，只能通过不可达节点测试错误路径，无法快速验证字段默认值、validator 指纹排序、Turbine 层级语义和快速通道缺失判断。

自查还发现：脚本设置 `$ErrorActionPreference = "Stop"` 后，`Write-Error` 会提前终止，原本设计的用法错误退出码 `2` 可能变成 `1`，不利于 CI 区分“参数错误”和“业务一致性失败”。

### 优化内容

- `tools/check_posnode_consistency.ps1` 新增 `-SelfTest`。
- 自检覆盖：
  - 缺失字符串字段返回默认值。
  - 非法 uint64 返回默认值。
  - `turbine_layer=-1` 使用有符号读取，不被错误转成 uint64。
  - validator 指纹按 `validator_id` 排序后稳定。
  - 缺少 `transaction_fast_path` 转为业务错误。
- `tools/run_posnode_gate.ps1` 和 `tools/check_posnode_consistency.ps1` 新增 `Stop-WithUsageError`。
- 用法错误统一写 stderr 并返回退出码 `2`。

### 复测结果

命令：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -SelfTest
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\run_posnode_gate.ps1 -SkipRace -RequireRace; "exit=$LASTEXITCODE"
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -Rounds 0 -RpcUrls http://127.0.0.1:1; "exit=$LASTEXITCODE"
```

关键输出：

```text
consistency self-test passed
exit=2
-SkipRace and -RequireRace cannot be used together
exit=2
Rounds must be greater than or equal to 1
```

结论：第五轮通过，巡检工具具备离线自检能力，用法错误退出码稳定为 `2`。

## 最终本机门禁

### 已执行命令

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\run_posnode_gate.ps1 -SkipRace
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -SelfTest
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -Help
```

关键输出：

```text
==> go test .\...
ok  	solana_golang	0.714s
ok  	solana_golang/cmd/posnode	(cached)
ok  	solana_golang/consensus	(cached)
ok  	solana_golang/runtime	(cached)
==> go vet .\...
==> go build .\cmd\posnode
WARNING: race gate skipped by -SkipRace. Production release must run go test -race on Linux or macOS.
consistency self-test passed
posnode.exe=False
posnode=False
```

### 结论

本机代码门禁和工具自检已闭环。真实生产结论仍必须等待以下数据：

- Linux/macOS 上执行 `go test -race ./...` 并归档输出。
- 5 台机器、每台 5 个节点的 RPC 地址文件。
- 至少 30 分钟、2 小时、24 小时三档一致性采样。
- 断网、延迟、乱序、节点重启、缺块同步、leader 不在线、fork/reorg 的故障注入报告。

当前版本可以作为生产候选前的本机验证版本，不能仅凭本机测试声明生产级完全通过。
