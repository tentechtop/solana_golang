# 十轮测试驱动优化报告

## 测试范围

本次是在前五轮基础上继续做 10 轮测试驱动优化，重点放在生产门禁脚本和多节点一致性巡检脚本。真实多机节点没有在本机直接启动，因此本报告不替代 5 机/30 节点长稳报告。

## 第1轮：非法 RPC URL 预校验

### 原始测试

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -RpcUrls not-a-url -Rounds 1 -TimeoutSeconds 1; "exit=$LASTEXITCODE"
```

原始问题：

```text
not-a-url snapshot failed: 操作超时
exit=1
```

### 优化

新增 `Normalize-RpcUrl`，在采样前校验 URL 必须是绝对 `http` / `https` 地址。

### 复测

```text
exit=2
Invalid RpcUrl: not-a-url
```

结论：非法 URL 现在属于用法错误，CI 可用退出码 `2` 识别。

## 第2轮：RPC URL 文件缺失

### 原始测试

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -RpcUrlFile .\report\missing-rpc-urls.txt -Rounds 1 -TimeoutSeconds 1; "exit=$LASTEXITCODE"
```

原始问题：

```text
Get-Content : Cannot find path ...
exit=1
```

### 优化

读取文件前使用 `Test-Path -LiteralPath` 检查，缺失时通过统一用法错误出口返回。

### 复测

```text
exit=2
RpcUrlFile does not exist: .\report\missing-rpc-urls.txt
```

结论：缺失 RPC 文件不再暴露底层 PowerShell 异常。

## 第3轮：逗号分隔 RPC URL

### 原始测试

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -RpcUrls "http://127.0.0.1:1,http://127.0.0.1:1/" -Rounds 1 -TimeoutSeconds 1
```

原始问题：

```text
Cannot convert value "http://127.0.0.1:1,http://127.0.0.1:1/" to type "System.Uri"
```

### 优化

新增 `Split-RpcUrlValue`，让帮助文档中的逗号分隔写法真实可用，并通过 URL 规范化去重。

### 复测

```text
==> consistency round 1/1
http://127.0.0.1:1/ snapshot failed: 操作超时
exit=1
consistency self-test passed
```

结论：逗号分隔输入已可用，重复根路径会规范化为同一个 URL。

## 第4轮：巡检 JSON 根摘要

### 原始测试

```powershell
$json = Get-Content -Path .\report\十轮测试输出\round3-after.json -Raw | ConvertFrom-Json
"hasNodeCount=$($null -ne $json.NodeCount)"
```

原始问题：

```text
hasNodeCount=False
```

### 优化

JSON 根对象新增：

- `NodeCount`
- `RoundCount`
- `SuccessfulSnapshotCount`
- `FailedSnapshotCount`
- `ErrorCount`
- `WarningCount`

### 复测

```text
NodeCount=1
RoundCount=1
FailedSnapshotCount=1
ErrorCount=2
```

结论：巡检报告可直接被 CI 和运维系统读取，不必再解析错误文本推断结果。

## 第5轮：失败节点结构化记录

### 原始问题

失败节点只存在于 `Errors` 字符串中，没有结构化字段记录 URL、轮次、时间和错误原因。

### 优化

新增：

- 根级 `FailedSnapshots`
- 每轮 `FailedSnapshots`

每条失败记录包含：

- `Round`
- `Url`
- `Error`
- `Timestamp`

### 复测

```powershell
$json = Get-Content -Path .\report\十轮测试输出\round4-round5-after.json -Raw | ConvertFrom-Json
"FailedSnapshots=$(@($json.FailedSnapshots).Count)"
```

关键输出：

```text
FailedSnapshots=1
```

结论：断网、节点重启、leader 离线等失败样本现在可结构化归档。

## 第6轮：长稳巡检快速失败

### 原始测试

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -RpcUrls http://127.0.0.1:1/ -Rounds 2 -IntervalSeconds 0 -TimeoutSeconds 1 -OutputPath .\report\十轮测试输出\round6-before.json
```

原始问题：

```text
==> consistency round 1/2
==> consistency round 2/2
```

### 优化

新增 `-FailFast`。当任一轮出现采样失败或一致性错误时，立即停止后续轮次。

### 复测

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -RpcUrls http://127.0.0.1:1/ -Rounds 2 -IntervalSeconds 0 -TimeoutSeconds 1 -FailFast -OutputPath .\report\十轮测试输出\round6-after.json
```

关键输出：

```text
==> consistency round 1/2
WARNING: round 1: fail fast stopped remaining rounds
RoundCount=1
WarningCount=1
```

结论：长稳测试可以按需快速失败，节省定位时间。

## 第7轮：生产门禁 JSON 输出

### 原始测试

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\run_posnode_gate.ps1 -Help -OutputPath .\report\十轮测试输出\round7-before.json; "exit=$LASTEXITCODE"
```

原始问题：

```text
exit=0
```

但没有生成结构化门禁报告。脚本也没有启用高级参数绑定，未知参数存在被静默忽略的风险。

### 优化

- `tools/run_posnode_gate.ps1` 增加 `[CmdletBinding()]`。
- 增加 `-OutputPath`。
- 新增 `Write-GateSummary`。
- `go test`、`go vet`、`go build` 每一步记录 `ExitCode`、`DurationMs`、`OK`。

### 复测

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\run_posnode_gate.ps1 -Help -OutputPath .\report\十轮测试输出\round7-gate-help.json
```

关键输出：

```text
ExitCode=0
Message=help
Steps=0
```

结论：门禁脚本具备 CI 可采集的 JSON 输出能力。

## 第9轮：RPC URL 文件行内注释

### 原始风险

真实 30 节点 URL 文件通常会写节点名或机器 IP 注释，例如：

```text
http://127.0.0.1:1 # node-1
```

原脚本只支持整行注释，不支持行内注释。

### 优化

新增 `Remove-InlineComment`，文件读取时先剥离 `#` 后的注释，再执行逗号拆分、URL 校验和规范化去重。

### 复测

测试文件：

```text
http://127.0.0.1:1 # local unreachable node
http://127.0.0.1:1/ # duplicate after normalization
```

关键输出：

```text
NodeCount=1
Url=http://127.0.0.1:1/
```

结论：RPC URL 文件可以带行内注释，重复根路径会被规范化去重。

## 第10轮：最小节点数门禁

### 原始风险

真实 5 机/30 节点测试中，如果 URL 文件少写节点，旧脚本仍会按已有节点运行，可能产生不完整的“通过”结论。

### 优化

新增 `-MinNodeCount`：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -RpcUrlFile .\rpc_urls.txt -MinNodeCount 30
```

当规范化去重后的 URL 数量不足时，直接返回用法错误退出码 `2`。

### 复测

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\check_posnode_consistency.ps1 -RpcUrls http://127.0.0.1:1/ -MinNodeCount 2 -Rounds 1 -TimeoutSeconds 1; "exit=$LASTEXITCODE"
```

关键输出：

```text
exit=2
RpcUrl count 1 is less than MinNodeCount 2
```

结论：真实多节点测试可以强制要求节点数量，避免漏测。

### 总结

本次 10 轮优化关闭了输入校验、报告结构、快速失败、CI JSON 输出、耗时统计、URL 文件可维护性和最小节点数门禁等问题。

仍未关闭的生产前硬门禁：

- Linux/macOS 上执行并归档 `go test -race ./...`。
- 使用 5 台机器、每台 5 个节点的真实 RPC URL 文件执行 `-MinNodeCount 30` 巡检。
- 跑满 30 分钟、2 小时、24 小时一致性采样。
- 执行断网、延迟、乱序、节点重启、缺块同步、leader 不在线、fork/reorg 故障注入。
